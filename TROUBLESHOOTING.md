# WEBTECH – פתרון תקלות (פנימי למפתח)

## 1. שגיאת Console: `webpage_content_reporter.js ... Unexpected token 'export'`

### משמעות
השגיאה מציינת שקובץ JS נטען כ-script רגיל (`<script src="...">` בלי `type="module"`) אבל מכיל `export` (ES Module).

### אבחון
| צעד | מה לבדוק |
|-----|-----------|
| 1 | בדוק ב-**Incognito** (ללא תוספי דפדפן) – האם השגיאה נעלמת? |
| 2 | חפש את הקובץ ב-`dist/` אחרי `npm run build` – האם `webpage_content_reporter.js` קיים? |
| 3 | השם "webpage_content_reporter" – מזכיר תוסף דפדפן / כלי בדיקה / סקריפט מוזרק, לא קובץ מהפרויקט |

### סיכום
- **אם הקובץ לא קיים ב-dist** → השגיאה כנראה מתוספת דפדפן. אין צורך בתיקון קוד.
- **פעולה:** הנחיות ללקוח – לבדוק באינקוגניטו או בדפדפן נטול תוספים.
- **אם הקובץ כן קיים ב-dist** → בעיית bundling/הזרקה – צריך להסיר או לטעון כ-module.

## 2. Vercel NOT_FOUND (fra1 / 404)

### משמעות
Vercel מחזיר 404 (NOT_FOUND) כשנכנסים ל-URL כלשהו.

### אבחון

**A) האם `/` (שורש) עובד?**
- אם `/` עובד אבל `/משהו` לא – זה בדרך כלל **SPA fallback** שחסר.
- **פתרון:** `vercel.json` עם `rewrites` – כל נתיב שאינו קובץ סטטי יחזיר `index.html`.

**B) מה ה-URL המלא של הלקוח?**
- וודא שהלקוח נכנס ל-`https://webtech-ochre.vercel.app/` (ללא נתיב פנימי).
- האתר הוא SPA – אין routes פנימיים. כל הניווט קורה בתוך העמוד.

**C) האם Assets (CSS/JS) נטענים?**
- פתח DevTools → Network.
- אם `style.css` או קבצי JS מחזירים 404 → בעיית `base` או Output Directory.
- **פתרון:** `vercel.json` עם `outputDirectory: "dist"` ותצורת Vite נכונה.

### מה תוקן
- נוסף `vercel.json` עם:
  - `outputDirectory: "dist"`
  - `buildCommand: "npm run build"`
  - `rewrites: [{ "source": "/(.*)", "destination": "/index.html" }]`

## 3. "האתר נראה הרוס" / בלי עיצוב

### סיבות אפשריות
1. **Cache** – הלקוח רואה גרסה ישנה.
2. **לינק לא נכון** – Preview ישן / דומיין שונה.
3. **נתיב פנימי** – נכנס ל-`/xyz` שמחזיר 404, ואז הדף ריק/שבור.
4. **CSS לא נטען** – 404 על קבצי assets.

### אבחון לפי סדר
1. וודא שהלקוח נכנס ל-URL הפרודקשן הנכון (לא Preview).
2. וודא שהוא לא על URL פנימי (רק `/`).
3. בדוק Network – אין 404 על CSS/JS.
4. נסה Hard Refresh: Ctrl+Shift+R (או Cmd+Shift+R ב-Mac).
