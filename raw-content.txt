שלב 1: להבין את הקונספט (SSR + SPA) ב-Next.js
חלק 1: מה זה “Hybrid” בכלל?

ב-Next.js אתה לא חייב לבחור “רק SPA” או “רק SSR”.
אתה יכול לבנות אתר שבו:

דפים מסוימים נטענים מהשרת עם HTML מוכן (SSR) = טוב ל-SEO ומהירות טעינה ראשונית.

חלקים אינטראקטיביים עובדים כמו אפליקציה בתוך הדפדפן (SPA/Client) = חוויה חלקה, בלי רענונים מיותרים.

במילים פשוטות:
השרת עושה את העבודה הכבדה בהתחלה, והדפדפן ממשיך את החוויה כמו אפליקציה.

חלק 2: הטריק הגדול של Next.js (Server Components מול Client Components)

ב-Next.js (App Router) ברירת המחדל היא Server Components:

רץ בשרת

מביא דאטה בקלות

שולח פחות JavaScript לדפדפן (יותר מהיר)

וכשצריך אינטראקטיביות (כפתורים עם state, טפסים חכמים, אנימציות, useEffect וכו’) — משתמשים ב-Client Components עם "use client".

זה לב ה-Hybrid.

חלק 3: איך זה ייראה באתר פורטפוליו שלך (דוגמה)

באתר שלך:

דף Home / Projects: רובו יהיה Server (מהיר + SEO).

כרטיס פרויקט: יכול להיות Server.

חיפוש/פילטר/סורט מיידי: Client (כמו SPA).

טופס יצירת קשר עם ולידציה/Toast: Client.

אם בעתיד תוסיף Dashboard / Admin: יכול להיות SSR או שילוב עם פעולות שרת.

משימת בדיקה קטנה (רק לוודא שהבנת)

תענה לי במשפט אחד:
מה ההבדל בין Server Component ל-Client Component?
---
המשפט המדויק לזכור

Server Component = להצגת מידע מהר וביעילות
Client Component = לאינטראקטיביות וחוויית משתמש

משהו חשוב מאוד שתדע

חברות גדולות (Netflix, Facebook וכו'):
לא משתמשות רק ב-Client.

הן משתמשות בדיוק בגישה הזו:
Hybrid = Server + Client ביחד

כלומר:

כמה שיותר Server (ביצועים)

Client רק איפה שצריך (אינטראקטיביות)
---
הכלל הכי חשוב ב-Hybrid (תשמור אותו)

אם הקומפוננטה:

רק מציגה מידע → Server

יש בה state / events / hooks → Client

זה כל הסיפור.

שלב 3: SSR ו-Data Fetching ב-Next.js
חלק 1: מה זה SSR בפועל?

SSR = Server Side Rendering

כלומר:

המשתמש נכנס ל-URL

השרת:

מביא נתונים (API / Database)

בונה HTML מוכן

הדפדפן מקבל דף מוכן

יתרונות:

טעינה ראשונית מהירה

SEO מצוין

פחות JavaScript בדפדפן

ב-Next.js זה קורה אוטומטית כשעושים fetch בתוך Server Component.

חלק 2: איך מביאים נתונים ב-Server Component

דוגמה:

// app/projects/page.tsx

async function getProjects() {
  const res = await fetch("https://api.example.com/projects");
  return res.json();
}

export default async function ProjectsPage() {
  const projects = await getProjects();

  return (
    <div>
      <h1>Projects</h1>
      {projects.map((project: any) => (
        <div key={project.id}>{project.title}</div>
      ))}
    </div>
  );
}


שימו לב:

הפונקציה של הדף היא async

אין "use client"

זה רץ בשרת

הדפדפן מקבל HTML מוכן

זה SSR.

חלק 3: איך זה הופך ל-Hybrid

עכשיו משלבים Client בתוך Server.

דוגמה:

Server Page:

import ProjectSearch from "./ProjectSearch";

export default async function ProjectsPage() {
  const projects = await fetchProjects();

  return (
    <div>
      <h1>Projects</h1>
      <ProjectSearch projects={projects} />
    </div>
  );
}


Client Component:

"use client";

import { useState } from "react";

export default function ProjectSearch({ projects }: any) {
  const [query, setQuery] = useState("");

  const filtered = projects.filter((p: any) =>
    p.title.toLowerCase().includes(query.toLowerCase())
  );

  return (
    <>
      <input onChange={(e) => setQuery(e.target.value)} />
      {filtered.map((p: any) => (
        <div key={p.id}>{p.title}</div>
      ))}
    </>
  );
}
